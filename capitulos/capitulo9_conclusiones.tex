\section{Conclusiones generales}

El desarrollo del \enquote{Asistente de Preparación \gls{pmp}} ha permitido extraer conclusiones significativas en múltiples dimensiones: técnica, pedagógica y estratégica.

\subsection{Conclusiones técnicas: la arquitectura como facilitador}
La decisión de utilizar una arquitectura moderna y desacoplada demostró ser clave para el éxito del proyecto.
\begin{enumerate}
\item \textbf{Baja latencia:} al procesar la información mediante streaming en el servidor, se logró una respuesta en tiempo real (promedio de 0.38 segundos). Esto confirma que, para la experiencia de usuario, la percepción de fluidez es crítica.
\item \textbf{Desacoplamiento del modelo:} el diseño permite sustituir el proveedor de IA (actualmente Google Gemini) por otro en el futuro sin impactar el resto del sistema. Esto mitiga el riesgo de \enquote{vendor lock-in} y permite adoptar modelos más avanzados conforme surjan.
\item \textbf{Eficiencia en la capa de datos:} la base de datos elegida ofrece alto rendimiento y bajo mantenimiento, vital para la escalabilidad del proyecto sin requerir un equipo de operaciones dedicado.
\end{enumerate}

\subsection{Conclusiones pedagógicas: aprendizaje dinámico}
La validación con usuarios (Capítulo 6) confirmó que la interacción conversacional es más efectiva que el estudio pasivo.
\begin{enumerate}
\item \textbf{Adaptabilidad contextual:} el sistema demostró capacidad para ajustar su rol (de mentor a auditor) según el contexto del estudiante. Esto valida el uso de IA para simulaciones de escenarios profesionales.
\item \textbf{Gamificación efectiva:} las métricas de progreso y niveles no son solo estéticas; resultaron funcionales para la retención del usuario y la reducción de la ansiedad frente a la magnitud del temario \gls{pmp}.
\end{enumerate}

\subsection{Conclusiones económicas y de negocio}
\begin{enumerate}
\item \textbf{Viabilidad económica:} el costo de mantener el sistema funcionando es muy bajo (menos de 50 centavos de dólar por usuario al mes). Esto hace posible ofrecer una versión gratuita para muchos estudiantes y sostener el negocio con una versión de pago.
\item \textbf{Desarrollo acelerado:} el uso de herramientas modernas permitió construir este software en 4 meses, demostrando cómo la tecnología actual multiplica la productividad de los desarrolladores.
\end{enumerate}

\section{Lecciones aprendidas y desafíos superados}

Crear este sistema tuvo sus retos. Aquí explicamos los más importantes y cómo los resolvimos.

\subsection{Normalización de datos y rendimiento}
Al inicio, la estructura de datos monolítica (chat, progreso, examen en un solo objeto) generaba cuellos de botella.
\begin{itemize}
\item \textbf{Lección:} el diseño de base de datos impacta directamente en la latencia de la aplicación.
\item \textbf{Solución:} desacoplamiento de entidades. El historial de chat se gestiona de forma efímera para maximizar velocidad, mientras que el progreso académico se persiste asíncronamente.
\end{itemize}

\subsection{Ingeniería de prompts (prompt engineering)}
En las primeras pruebas, el modelo tendía a ser demasiado permisivo, afectando el rigor pedagógico.
\begin{itemize}
\item \textbf{Lección:} un modelo de lenguaje generalista requiere instrucciones de sistema (system prompts) muy precisas para roles especializados.
\item \textbf{Solución:} iteración sobre los prompts para definir límites claros (\enquote{guardrails}) para instruir al modelo a actuar como auditor socrático y no como simple buscador de respuestas.
\end{itemize}

\subsection{Gestión de streaming y \gls{ux}}
La renderización de texto carácter por carácter (streaming) presentó desafíos de sincronización en el frontend.
\begin{itemize}
\item \textbf{Lección:} la experiencia de usuario en tiempo real requiere un manejo robusto del estado en el cliente.
\item \textbf{Solución:} implementación de hooks personalizados en el frontend para gestionar el flujo de tokens entrantes sin bloquear la interfaz de usuario.
\end{itemize}

\section{Aportes originales del trabajo}

Este proyecto aporta nuevas perspectivas sobre la integración de \gls{ia} generativa en entornos educativos:

\begin{enumerate}
\item \textbf{Orquestación de roles dinámicos:} metodología para cambiar el comportamiento del agente (de tutor a evaluador) en tiempo de ejecución mediante inyección de contexto, sin reentrenamiento.
\item \textbf{Hibridación pedagógica:} fusión de la evaluación estructurada (exámenes tipo test) con la flexibilidad del diálogo abierto (chat con IA).
\item \textbf{Latencia como factor crítico de aprendizaje:} evidencia de que la baja latencia correlaciona positivamente con la inmersión del estudiante en sesiones de estudio conversacional.
\end{enumerate}
